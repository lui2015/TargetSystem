## 1. 文档概述

### 1.1 编写目的

定义一款面向个人用户的**目标管理系统**的产品需求，通过融合前沿目标管理方法论（OKR、SMART、WOOP、GTD、原子习惯等），帮助用户完成「**科学制定 → 高效执行 → 持续打卡 → 周期复盘 → 迭代优化**」的闭环，解决个人目标「定不准、做不到、坚持不下去、复盘不深入」四大核心痛点。

### 1.2 术语定义

| 术语 | 含义 |
| --- | --- |
| Objective（O） | 定性的、有方向感的长期目标 |
| Key Result（KR） | 可量化的关键结果，用于衡量 O 的达成度 |
| SMART | Specific / Measurable / Achievable / Relevant / Time-bound |
| WOOP | Wish / Outcome / Obstacle / Plan，心理对比与执行意图工具 |
| Habit Loop | Cue（提示）→ Craving（渴求）→ Response（行动）→ Reward（奖励） |
| Check-in | 打卡：对目标/习惯完成情况的单次记录 |
| Streak | 连续打卡天数 |

## 2. 产品定位与目标

### 2.1 产品愿景

> **"让每一个认真对待人生的人，都能把目标变成现实。"**

做一款"带着方法论的目标陪跑工具"，区别于 Todo / 日程类产品，核心强调**目标的科学制定**与**长期坚持**。

### 2.2 产品目标（可量化）

| 维度 | 目标 |
| --- | --- |
| 用户价值 | 用户使用 3 个月后，目标完成率相比使用前提升 ≥ 40% |
| 留存 | 次日留存 ≥ 45%，30 日留存 ≥ 25% |
| 习惯粘性 | 平均每用户每周打卡 ≥ 4 次 |
| NPS | ≥ 40 |

### 2.3 差异化竞争力

- 内置**方法论引导式**目标制定流程（而非空白编辑器）。
- 目标-习惯-打卡三层结构，解决"大目标悬浮"的痛点。
- **AI 教练**：基于复盘数据给出个性化调整建议。
- 强复盘闭环（周/月/季/年），复盘即成长。

## 3. 用户画像与场景

### 3.1 典型用户画像

**P1：职场进阶者（主要目标用户）**
- 25–35 岁，互联网/金融/咨询从业者。
- 痛点：目标多但散、执行靠意志力、年底才发现年初目标没达成。
- 诉求：结构化管理 OKR、跟踪进度、复盘效率。

**P2：自我提升型学生**
- 18–24 岁，大学生/考研考证人群。
- 痛点：学习计划执行差、容易拖延、缺少即时正反馈。
- 诉求：清晰的任务拆解、打卡激励、学习社群陪伴。

**P3：生活习惯重塑者**
- 健身、阅读、冥想、戒烟戒酒等习惯养成人群。
- 诉求：长期坚持、可视化进度、连续打卡奖励。

### 3.2 核心使用场景

| 场景 ID | 场景描述 |
| --- | --- |
| S1 | 年初/季度初，用户制定年度 OKR，系统引导完成 O→KR→里程碑→周计划拆解 |
| S2 | 每天早晨打开 App，查看今日要做的关键动作与习惯打卡 |
| S3 | 每完成一项习惯（如跑步、阅读 30 分钟），一键打卡，获得连续天数激励 |
| S4 | 每周日晚进行周复盘，系统引导填写进度/障碍/下周调整 |
| S5 | 用户连续 3 周某 KR 进度低于预期，AI 教练提醒调整目标或方法 |

## 4. 方法论底座

系统不是简单的"任务清单"，而是把以下方法论**内嵌成流程与模板**：

| 阶段 | 方法论 | 系统落地 |
| --- | --- | --- |
| 目标制定 | **SMART** | 创建 KR 时校验是否可量化、有截止时间 |
| 目标制定 | **OKR** | 三层结构：Objective → Key Result → Initiative |
| 执行准备 | **WOOP** | 引导用户写出 Wish / Outcome / Obstacle / Plan（If-Then 执行意图）|
| 执行 | **原子习惯（Atomic Habits）** | 习惯最小化、提示-行动-奖励闭环、Habit Stacking |
| 执行 | **GTD** | 下一步行动（Next Action）、Inbox 收集 |
| 复盘 | **PDCA / KPT** | 周复盘模板：Keep / Problem / Try |
| 心态 | **成长型思维** | 复盘引导"从失败中学到了什么"而非"是否完成" |

### 5.2 模块一：目标管理（P0）

#### FR-G-01 目标创建：方法论引导式向导

**需求描述**：用户新建目标时，不提供空白输入框，而是提供**分步引导向导**。

**流程**：

1. **选择目标类型**：职业 / 学习 / 健康 / 财务 / 关系 / 兴趣 / 其他
2. **书写 Objective**：
   - 引导语："用一句话描述你希望在[周期]内达到的状态"
   - 示例提示、字数校验（10–50 字）
3. **拆解 Key Results（2–5 条）**：
   - 每条 KR 强制包含：**度量指标 + 目标值 + 截止日期**
   - 系统进行 **SMART 校验**，不满足给出红色提示（如"未包含数字/单位"）
4. **WOOP 预案**：
   - Wish：我想……
   - Outcome：达成后最大的好处是……
   - Obstacle：最可能阻碍我的是……
   - Plan：**当 [障碍] 发生时，我就 [替代行动]**（If-Then）
5. **生成目标卡片**，进入目标看板。

**验收标准**：
- 未填写 KR 或 KR 不符合 SMART 时无法进入下一步。
- 系统为每个目标类型提供 ≥ 3 个示例模板。

#### FR-G-02 目标拆解（OKR → 里程碑 → 周计划 → 日任务）

- 支持四层结构：**Objective → Key Result → Milestone → Task**。
- 支持从 KR 一键"拆解为里程碑"（AI 辅助，用户可编辑）。
- 每个 Task 可挂载到具体日期，自动出现在"今日待办"。

#### FR-G-03 目标看板

- **视图 1：OKR 卡片视图**——卡片展示 O、KR 进度条、剩余天数。
- **视图 2：树状结构视图**——从 O 到 Task 的层级展开。
- **视图 3：时间轴视图**——甘特图样式，展示里程碑。
- 支持按周期（季/年）、类型、状态筛选。

#### FR-G-04 进度更新

- KR 进度支持三种更新方式：
  1. **手动填数值**（如已跑 50 / 100 公里）。
  2. **从关联 Task 自动聚合**（任务完成数/总数）。
  3. **从关联习惯打卡自动聚合**（连续 30 天打卡 = 100%）。

### 5.3 模块二：习惯与打卡（P0 · 核心）

> 这是帮助用户"坚持奋斗"的核心模块。

#### FR-H-01 习惯创建

**字段**：
- 习惯名称、图标、颜色
- 关联目标（可选，强烈推荐）
- **频率**：每日 / 每周 N 次 / 指定星期 / 自定义周期
- **时段**：早/中/晚/自定义时间
- **难度**：简单 / 中等 / 困难（影响经验值）
- **最小行动（Atomic 化）**：如"每天阅读 2 页书"
- **Habit Stacking**：绑定到一个已有习惯之后触发（如"刷完牙之后，冥想 1 分钟"）
- **提醒**：时间提醒 + 地点提醒（可选）
- **奖励**：完成后的自我奖励描述（可选）

#### FR-H-02 每日打卡

- 主屏"今日习惯"列表，一键点击完成。
- 支持 3 种打卡类型：
  - **布尔型**（完成/未完成）
  - **计数型**（做了 N 次 / 目标 M 次）
  - **计时型**（专注了 N 分钟）
- 打卡后立即反馈：动画 + 经验值 + Streak 天数更新 + 鼓励文案。
- 支持添加心情（😄/😐/😫）与简短笔记。

#### FR-H-03 连续天数（Streak）机制

- 计算规则：按习惯设定频率计算"应打卡日"，连续未断 = Streak +1。
- **护盾机制**：每月赠送 1–2 张"忘记卡"，允许补打前一天。
- 里程碑庆祝：Streak 达到 7 / 21 / 66 / 100 / 365 天时全屏庆祝动画。
- 断签提醒："你已连续 N 天，别让今天中断。"

#### FR-H-04 打卡日历与热力图

- GitHub 风格的年度热力图，一眼看出坚持情况。
- 周视图、月视图支持切换。
- 点击日期查看当日所有打卡明细。

#### FR-H-05 补卡与宽容度

- 支持 24 小时内补卡（标记为"补卡"，不计入完美连续）。
- 允许用户配置"允许每周跳过 N 次"不算断签（防完美主义挫败感）。

### 5.4 模块三：任务执行（P0）

#### FR-T-01 今日视图（Today）

- **三栏布局**：今日习惯 | 今日任务 | 今日专注时间
- 顶部显示：今日目标进度摘要、激励语。
- 支持从"Inbox"快速拖入今日。

#### FR-T-02 番茄专注（P1）

- 25/5 经典番茄钟，可自定义。
- 专注时可锁定选择"服务于哪个 KR"，结束后自动记入该 KR 的投入时长。
- 中断记录（次数 + 原因），用于复盘分析。

#### FR-T-03 Inbox 快速收集（GTD）

- 顶部悬浮按钮，随时记录一闪而过的想法/任务，稍后整理归类到目标下。

### 5.5 模块四：复盘系统（P0）

> **"不复盘的执行等于没执行。"** 系统通过模板化引导降低复盘门槛。

#### FR-R-01 周复盘（每周日自动弹出）

**模板（KPT + 数据）**：
1. 本周数据自动生成：习惯完成率、KR 进度变化、专注总时长、断签次数。
2. **Keep**：本周做得好的三件事。
3. **Problem**：遇到的最大障碍是什么？
4. **Try**：下周要尝试的一个小改变。
5. 下周重点 KR 选择（最多 3 个）。

#### FR-R-02 月/季/年复盘（P1）

- 更深度模板，包含情绪曲线、精力分布、目标调整。
- 季度复盘支持 OKR 打分（0.0–1.0），过低建议拆解或放弃。

#### FR-R-03 AI 教练（P1）

- 基于用户数据给出个性化洞察：
  - "你在周三/周四 KR 进度贡献最大，建议将高难度任务安排在此。"
  - "这个 KR 连续 3 周进度 <20%，可能目标过大，建议拆解。"
  - "阅读习惯断签集中在加班日，是否调整为'到家后立刻阅读 5 分钟'？"
- 提供但不强制，用户可接受/忽略建议。

#### FR-R-04 复盘历史

- 所有复盘按时间线归档，支持搜索、重读、对比。

---

### 5.6 模块五：激励系统（P1）

#### FR-M-01 成就徽章

- 预置徽章：首次打卡、7 日连击、100 小时专注、完成第一个 KR 等。
- 徽章墙可在个人主页展示。

#### FR-M-02 等级与经验值（XP）

- 完成习惯/任务/复盘获得 XP，难度越高 XP 越多。
- 等级与称号：新手 → 践行者 → 守护者 → 大师。

#### FR-M-03 数据可视化

- 年度总结报告（类似"网易云年度报告"）。
- 多维图表：习惯趋势、KR 燃尽图、精力热力图。

## 6. 非功能需求

| 维度 | 要求 |
| --- | --- |
| 性能 | 首屏打开 ≤ 1.5s；打卡响应 ≤ 200ms |
| 可用性 | 核心功能 SLA ≥ 99.9% |

| 国际化 | 首版简体中文，预留 i18n 框架 |

## 7. 数据模型（核心实体概览）

```
User ──┬── Objective ──┬── KeyResult ──┬── Milestone ── Task
       │               │               └── Initiative
       │               └── WOOPPlan
       │
       ├── Habit ──── CheckIn ──── Streak
       │
       ├── Review（Weekly / Monthly / Quarterly / Yearly）
       │
       └── Achievement / XPLog / Setting
```

关键字段示意（以 Habit / CheckIn 为例）：

- **Habit**：id, userId, name, goalId?, frequencyRule, type(bool/count/timer), targetValue, reminderRule, stackAfterHabitId?, createdAt
- **CheckIn**：id, habitId, userId, checkDate, value, mood, note, isMakeUp, createdAt
- **Streak**：habitId, currentStreak, longestStreak, lastCheckDate, shieldCount

> ⚠️ 数据库访问层必须 100% 使用 ORM 参数绑定或预编译 SQL，禁止任何形式的字符串拼接。

## 8. 关键用户流程（Happy Path）

### 8.1 新用户首启

```
启动 → 引导页（方法论介绍）→ 注册登录 →
制定第一个目标（向导，5 步）→ 拆解出 1 个习惯 →
跳转到 Today 视图，看到"第一次打卡"按钮高亮
```

### 8.2 每日核心路径

```
打开 App → Today 视图 → 完成 3 件事：
  ① 打卡习惯  ② 番茄专注一次  ③ 勾选一个任务 →
晚间轻量回顾（情绪+一句话总结）
```

### 8.3 每周复盘路径

```
周日晚 21:00 推送 → 进入复盘向导 →
看到本周数据自动汇总 → 填写 KPT（3 分钟）→
选定下周 3 个重点 KR → 保存并生成复盘卡片可分享
```

## 10. 风险与对策

| 风险 | 影响 | 对策 |
| --- | --- | --- |
| 功能过重导致学习成本高 | 新用户流失 | MVP 做减法；引导式首启；"极简模式"开关 |
| 用户打卡一段时间后疲劳 | 留存下降 | 动态难度、成就/报告惊喜、AI 教练主动关怀 |
| 方法论对普通用户太硬核 | 不敢用 | 提供丰富示例模板与情景化话术 |
| AI 建议不准/冒犯 | 信任崩塌 | AI 输出标注"仅供参考"；用户可反馈调优 |
| 数据隐私焦虑 | 用户不敢写真实目标 | 默认本地优先 + 端到端加密可选；隐私承诺透明化 |
